# StudyPal 项目结构说明

## 项目概述

StudyPal 是一个基于 Flask 的 Python 桌面应用，集成了 AI 学习助手、番茄钟、任务管理、成就系统等学习辅助功能。

## 目录结构

```
StudyPal/
├── app.py                    # Flask 主应用入口
├── config.py                 # 全局配置文件
├── requirements.txt          # Python 依赖包列表
├── README.md                 # 项目说明文档
├── .gitignore               # Git 忽略文件配置
│
├── data/                     # 数据存储目录
│   ├── ai_history.json       # AI 对话历史记录
│   ├── study_log.json        # 学习日志
│   ├── tasks.json            # 任务数据
│   └── user_settings.json    # 用户设置
│
├── docs/                     # 项目文档
│   └── StudyPal 项目文档.md
│
├── src/                      # 源代码目录
│   ├── ai/                   # AI 相关模块
│   │   ├── __init__.py

│   │
│   ├── core/                 # 核心功能模块
│   │   ├── __init__.py
│   │   ├── buddy.py          # 学习搭子核心逻辑
```

## 核心模块说明

### 1. 主应用 (app.py)

Flask 应用入口，负责：
- 路由定义
- API 端点
- 模板渲染
- 模块初始化

### 2. 配置 (config.py)

全局配置管理：
- 文件路径配置
- API 密钥设置
- 应用参数

### 3. AI 模块 (src/ai/)

#### ai_helper.py
- 提供 AI 问答功能
- 支持对话上下文
- 集成外部 AI API

### 4. 核心模块 (src/core/)

#### buddy.py
- 学习搭子状态管理
- 情绪/状态反馈
- 学习建议生成

#### timer.py
- 学习计时器
- 番茄钟功能
- 休息提醒

### 5. 功能模块 (src/modules/)

#### achievements.py
成就系统管理：
- 成就定义与解锁
- 积分与等级系统
- 成就进度追踪

#### ai_memory.py
AI 对话历史：
- 对话持久化
- 历史记录管理
- 对话搜索功能

#### data_manager.py
数据管理：
- 数据读写
- 数据备份
- 数据同步

#### plan_generator.py
学习计划生成：
- AI 生成计划
- 基础计划生成
- 计划任务管理

#### study_calendar.py
学习日历：
- 学习记录展示
- 统计图表
- 学习趋势分析

#### task_manager.py
任务管理：
- 任务 CRUD
- 截止日期提醒
- 任务优先级

## API 端点列表

| 端点 | 方法 | 描述 |
|------|------|------|
| `/` | GET | 主页面 |
| `/api/ask` | POST | AI 问答 |
| `/api/ai/history` | GET/POST | 对话历史列表/创建 |
| `/api/ai/history/<id>` | GET/DELETE | 对话详情/删除 |
| `/api/achievements` | GET | 成就数据 |
| `/api/stats` | GET | 统计数据 |
| `/api/notification/settings` | POST | 通知设置 |

## 成就系统

### 成就类型
1. **番茄钟成就** - 基于完成的番茄钟数量
2. **连续学习成就** - 基于连续学习天数
3. **学习时长成就** - 基于累计学习时长
4. **任务完成成就** - 基于完成任务数量
5. **AI 对话成就** - 基于提问/对话次数
6. **目标达成成就** - 基于达成每日目标次数

### 等级系统

## 开发指南

### 添加新功能
1. 在 `src/modules/` 创建新模块文件
2. 在 `app.py` 中添加对应的 API 路由
3. 在 `templates/index.html` 中添加前端界面
4. 更新本文档

### 修改数据格式
1. 修改 `config.py` 中的数据路径配置
2. 更新 `src/modules/data_manager.py` 中的数据读写逻辑
3. 确保数据迁移兼容性

## 常见问题

### Q: 如何修改 AI API 密钥？
A: 在 `config.py` 中修改 `API_KEY` 配置项

### Q: 如何更改每日学习目标？
A: 在前端设置中修改，或直接在 `user_settings.json` 中修改 `daily_goal` 值

### Q: 数据如何备份？
A: 定期复制 `data/` 目录到安全位置
